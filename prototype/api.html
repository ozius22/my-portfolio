01 — Foundation

Design Table Before you write any code

Always start by designing your database table on paper or a whiteboard. Define every column — its name, data type, constraints, and whether it's indexed or nullable.

⊞ customers InnoDB · utf8mb4
Column Type Nullable Default Notes
id BIGINT UNSIGNED NO AUTO_INCREMENT Primary Key
company_id BIGINT UNSIGNED YES NULL FK → companies.id
name VARCHAR(255) NO — Full name
email VARCHAR(255) NO — UNIQUE index
status ENUM NO 'active' active / inactive
created_at TIMESTAMP YES NULL Auto by Laravel
updated_at TIMESTAMP YES NULL Auto by Laravel

↑ Example: customers table. Design yours first, THEN generate the migration.

02 — Database

Migration

A migration is like a version control for your database. It creates, modifies, or drops tables — and can be rolled back if something goes wrong.

▸

Creates a file in database/migrations/

▸

Run with php artisan migrate

▸

Rollback with php artisan migrate:rollback

$ php artisan make:migration create_class_table

# → database/migrations/2024_01_01_000000_create_class_table.php

// Inside the migration file:

public function up(): void {

  Schema::create('classes', function (Blueprint $table) {

    $table->id();

    $table->string('name');

    $table->timestamps();

  });

}

03 — Database

Seeder

Seeders populate your database with initial or test data. Perfect for creating default admin users, lookup tables, or demo records — things that should exist on every fresh install.

$ php artisan make:seeder MaintenanceAdminSeeder

# → database/seeders/MaintenanceAdminSeeder.php

// Inside the seeder:

public function run(): void {

  User::create([

    'name' => 'Maintenance Admin',

    'email' => 'admin@system.com',

    'role' => 'maintenance',

  ]);

}

php artisan db:seed --class=MaintenanceAdminSeeder

04 — ORM Layer

Model

The Model is your Eloquent ORM class — it represents a database table and lets you interact with rows as PHP objects. You also define relationships here.

$ php artisan make:model Customer

# Creates: app/Models/Customer.php


$ php artisan make:model Company -m

# -m flag = also creates the migration file automatically ✓

// Eloquent Relationships

Method Meaning Example
hasOne() One-to-One Customer → Profile
hasMany() One-to-Many Company → Customers
belongsTo() Inverse of hasOne/hasMany Customer → Company
belongsToMany() Many-to-Many (pivot table) User ↔ Role
hasManyThrough() Through a middle model Country → Posts via Users

// app/Models/Customer.php

class Customer extends Model {

  protected $fillable = ['name', 'email', 'company_id'];


  public function company(): BelongsTo {

    return $this->belongsTo(Company::class);

  }

}

05 — HTTP Layer

Route RESTful API Resources

Route::apiResource() registers all 5 RESTful routes for a resource in a single line — no need to define each method separately.

// routes/api.php

Route::apiResource('customers', CustomerController::class);


// Multiple at once:

Route::apiResources([

  'customers' => CustomerController::class,

  'companies' => CompanyController::class,

  'policies' => PasswordPolicyController::class,

]);

apiResource('customers', CustomerController) → generates:
Method URI Action Route Name
GET /customers index customers.index
POST /customers store customers.store
GET /customers/{id} show customers.show
PUT /customers/{id} update customers.update
DELETE /customers/{id} destroy customers.destroy

06 — HTTP Layer

Controller

The controller handles incoming HTTP requests and returns a response. The --resource flag pre-generates all 5 method stubs automatically.

$ php artisan make:controller PasswordPolicyController --resource

# → app/Http/Controllers/PasswordPolicyController.php

// Pre-generated method stubs:

public function index() {} // GET /resource

public function store() {} // POST /resource

public function show() {} // GET /resource/{id}

public function update() {} // PUT /resource/{id}

public function destroy() {} // DELETE /resource/{id}

💡 Controller responsibility

The controller should be thin. It receives the request, delegates to a Service for business logic, and returns a Resource response. It does NOT contain business logic itself.

07 — HTTP Layer

Form Request

Use a Form Request class when your validation rules are complex — it keeps your controller clean and centralizes authorization + validation logic.

✗ Simple: inline $request->validate() ✓ Complex: FormRequest class

$ php artisan make:request CompanyStoreRequest

# → app/Http/Requests/CompanyStoreRequest.php

class CompanyStoreRequest extends FormRequest {


  public function authorize(): bool {

    return true; // or check auth policy

  }


  public function rules(): array {

    return [

      'name' => ['required', 'string', 'max:255'],

      'email' => ['required', 'email', 'unique:companies'],

      'tax_id' => ['nullable', 'string', Rule::unique('companies')],

    ];

  }

}

08 & 09 — Business Logic

Service & Repository

These two are not generated by artisan — you create them manually. They enforce separation of concerns and make your code testable.

S
⚙️

Service

app/Services/

Contains your business logic. Called by the controller. It orchestrates data operations but doesn't directly query the DB.

// app/Services/CustomerService.php

class CustomerService {

  public function create(array $data) {

    // validate, transform, then:

    return $this->repo->create($data);

  }

}

R
🗄️

Repository

app/Repositories/

Handles all data access (Eloquent queries). The service calls the repository — swapping DB logic doesn't affect business logic.

// app/Repositories/CustomerRepository.php

class CustomerRepository {

  public function create(array $data) {

    return Customer::create($data);

  }

  public function findById(int $id) {

    return Customer::findOrFail($id);

  }

}

Controller → Service → Repository → Model → Database Clean, testable, decoupled ✓

10 — Output Layer

API Resource

An API Resource transforms your Eloquent model into a JSON response. It lets you control exactly which fields to expose — hiding sensitive data and shaping the output cleanly.

$ php artisan make:resource PasswordPolicyResource

# → app/Http/Resources/PasswordPolicyResource.php

class PasswordPolicyResource extends JsonResource {

  public function toArray(Request $request): array {

    return [

      'id' => $this->id,

      'min_length' => $this->min_length,

      'requires_2fa'=> $this->requires_2fa,

      'updated_at' => $this->updated_at,

      // 'password' is NOT exposed — intentionally

    ];

  }

}

Single record

return new PasswordPolicyResource($policy);

Collection

return PasswordPolicyResource::collection($policies);
